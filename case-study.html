Project Deep Dive

To-Do List Application

Building a working to do list using HTML, CSS, and Vanilla JavaScript.

What Was the Brief or Problem You Were Solving?

The goal of this project was to build a working To-do List application using only HTML, CSS, and Vanilla JavaScript. Compared to the projects from last term, this one was more challenging because it focused a lot more on JavaScript and making the app interactive.

The app needed to let users:

  • Add tasks
  • Mark tasks as completed
  • Delete tasks
  • Filter tasks by status (All, Active, Completed)
  • Save tasks using localStorage so they stay even after refreshing the page

One of the biggest challenges was that we could only use MDN documentation for help. We were not allowed to use tutorials, YouTube videos, or Stack Overflow, so I had to learn how to solve problems mostly by reading documentation and experimenting on my own.

What Decisions Did You Make and Why?

I decided to keep the design clean and simple so the app would be easy to use. I also separated different JavaScript functions for adding, deleting, completing, and filtering tasks because it helped keep the code more organized and easier to understand.

I used event listeners to make the app interactive and localStorage to save the tasks after page reloads. I chose these methods because they matched the project requirements and helped me practice important JavaScript concepts.

What Tools and Technologies Did You Use and What Did You Learn?

For this project I used:

  • HTML
  • CSS
  • Vanilla JavaScript
  • MDN Web Docs

This project helped me understand JavaScript better, especially:

  • DOM manipulation
  • Event listeners
  • querySelector
  • createElement
  • localStorage

The MDN pages helped me a lot because they explained concepts clearly and gave examples I could test in my own project. It took time and effort to understand some parts, but I learned a lot through trial and error.

What Would You Do Differently & What Are You Proud Of?

What I Would Do Differently: If I worked on this project again, I would improve the design and make it more responsive on smaller screens. I would also organize my JavaScript code better and maybe add extra features like editing tasks or adding deadlines. I also think spending more time planning before coding would help me avoid some bugs and confusion later on.

What Are You Proud Of: I am proud that I managed to build a fully working To-do List app using only Vanilla JavaScript and MDN documentation. At first, some parts were difficult, especially localStorage and filtering tasks, but I kept working through the problems until I understood them. This project helped me become more confident with JavaScript and improved my problem-solving and debugging skills.
